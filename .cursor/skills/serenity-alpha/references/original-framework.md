# Serenity Alpha 原始框架

本参考文档以精简形式保留 `a.md` 中的源框架。

## 本质

不要问新闻是否够大。要问：

- 背后是否有已观察到的需求变化？
- 该需求能否进入公司财务报表？
- 哪家公司足够小，需求足以改变其命运？
- 市场是否误分类了该公司？
- 假设能否在 1-4 个季度内通过财报验证？

## 过滤条件

避开以下类型的新闻：

- 有情绪但无需求
- 有真实需求但仅大盘受益者
- 已充分定价的赢家
- 传导链过长
- 无财务验证点

## Alpha 公式

```text
潜在 alpha 强度 =
  需求确定性
  × 财务传导清晰度
  × 业务纯度
  × 市值弹性
  × 市场忽视度
```

对可验证的非共识想法使用小初始仓位，随后根据证据更新。
