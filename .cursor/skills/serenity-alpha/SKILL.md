# Serenity Alpha

## 核心原则

不要只问新闻是否足够震撼。要问：一个已经可观察到的需求变化，能否改写一家较小公司的财务报表。

使用本 skill 将新闻转化为可验证的 alpha 假设：

```text
新闻 -> 已发生的需求变化 -> 收入/利润传导 -> 小市值弹性 -> 验证路径
```

将输出视为研究假设，而非投资建议。在点名证券或做出时效性判断前，务必从可靠来源核实当前价格、市值、 filings、财报电话和新闻。

## 回答结构

以最符合 alpha 假设的公司作为开头和结尾。

- 开头用一句直接判断：`最值得优先验证的是：公司名 / ticker，原因是...`
- 若出现多个标的，选定一个主要候选，其余标注为备选或供应链可比公司。
- 若尚无足够明确的投资标的，在开头说明：`暂时没有足够明确的公司，先观察...`
- 结尾再次点名同一主要公司，并给出能确认或证伪该假设的单一验证条件。

## 工作流程

### 1. 区分叙事与需求

判断该信息仅是叙事，还是需求拐点。优先关注已经发生的事实：

- 用户正在采用或付费
- 企业正在采购或扩大部署
- 供应商正在出货、积压订单或提价
- 排产计划收紧
- 财报电话或 filings 提及该需求
- 细分生态变得拥挤

若需求不可观察，将想法归类为仅观察名单。

### 2. 将现象翻译为财务科目

使用以下句式：

```text
因为 X 正在发生，对 Y 的需求正在增加。
该需求可通过收入线 Z 流向 A/B/C 公司。
公司 N 可能重要，因为其市值、收入基数或业务纯度足够小，需求冲击足以改变报表结果。
```

对每个候选，将机制映射到利润表或资产负债表科目：

- 收入：销量、ASP、客户数、使用量、抽成率、 backlog 转化
- 毛利率：结构变化、定价权、产能利用率、投入成本
- 经营杠杆：固定成本摊薄、销售效率、研发杠杆
- 现金流：营运资本、capex 需求、库存周转、预付款

### 3. 优先小盘、高纯度、被误分类的标的

寻找「小铲子」，而非仅显而易见的大盘赢家。以下特征排名更高：

- 市值或收入基数相对需求冲击规模较小
- 对特定需求向量业务纯度高
- 处于关键供应链位置或具有稀缺价值
- 当前投资者关注度低
- 市场标签陈旧，未能反映新角色
- 1-4 个季度内可能获得财务验证

使用以下粗略筛选：

```text
alpha 弹性 ~= 增量需求影响 / 当前公司规模
```

### 4. 检验市场误分类

追问：

- 市场目前认为这家公司是什么？
- 若需求变化持续，它可能正在变成什么？
- 新标签是否足够大、足够持久、且足够接近报表数字以产生影响？

误分类产生 alpha 的前提是：重新标签化能被数字验证，而非仅靠故事。

### 5. 构建验证链

将假设转化为可观察的检查点。使用最相关的指标：

- 收入增长或指引修订
- 毛利率改善或结构披露
- backlog、book-to-bill、交期或订单评论
- 库存去化或渠道核查
- 产能利用率、扩产或 capex 计划
- ASP 变化或定价评论
- 客户集中度变化
- 管理层主动提及需求驱动因素
- 竞争对手/供应商/客户佐证

定义什么会确认、削弱或证伪该假设。

### 6. 评估 Alpha 强度

对每个候选进行定性或 1-5 分评分：

| 维度 | 问题 |
| --- | --- |
| 需求确定性 | 这是已发生，还是仅想象？ |
| 传导清晰度 | 需求能否清晰流向具名公司？ |
| 业务纯度 | 公司是否直接暴露？ |
| 市值弹性 | 公司是否足够小以让影响有意义？ |
| 市场忽视度 | 市场是否遗漏或误分类？ |
| 验证速度 | filings 或电话能否在 1-4 季度内验证？ |
| 下行风险 | 假设错误时会发生什么？ |

优先选择高确定性、传导清晰、高纯度、高弹性、可近期验证的想法。

### 7. 按证据定仓位

将仓位姿态表述为条件化研究指引，而非个性化推荐：

| 假设状态 | 姿态 |
| --- | --- |
| 需求似真，传导不明 | 观察 / 极小试探仓位 |
| 需求真实，传导清晰，尚无财务证明 | 若风险匹配可小仓试错 |
| 财务开始验证且市场仍低估 | 考虑加仓 |
| 假设已成共识且估值拉伸 | 降低回报预期 / 仅交易 |
| 关键假设被证伪 | 退出或移出观察名单 |

## 输出模板

分析新闻时使用以下结构：

```markdown
## 结论先行：优先验证的公司
点名最值得优先验证的一家公司 / ticker，并用一句话说明为什么它是本次新闻里最有弹性的候选。

## A. 表层新闻
简述新闻表面信息。

## B. 已发生的需求变化
说明已经可观察的需求、采购、使用、价格、排产或供应链变化；如果没有，明确说只是谈资。

## C. 财务翻译
把需求映射到收入项、成本项、利润项、现金流或资产负债表项目。

## D. 受益链条
列出一阶、二阶、三阶受益者，并说明传导距离。

## E. 小市值高弹性标的
列出可能的小盘/高纯度/低关注候选；标注需要核实的市值、收入基数和业务占比。

## F. 市场误分类
说明市场现在把公司当什么，它可能正在变成什么。

## G. 验证指标
列出未来 1-4 个季度要看的财报、电话会、供应链和价格指标。

## H. 下行风险
列出需求、传导、竞争、估值、时点和流动性风险。

## I. 仓位建议
给出观察、小仓、加仓、放弃的条件；避免承诺收益或给出个性化投资指令。

## 最后一句
再次点名这家公司，并给出最关键的财报验证条件；如果条件不成立，明确说应放弃或降级为观察。
```

## 质量标准

- 分析锚定在可观察需求，而非情绪。
- 优先具名收入机制，而非宽泛主题。
- 区分一阶受益者与遥远的二、三阶故事。
- 使用当前来源数据核实市值、财务、价格和近期 filings。
- 明确说明不确定性和证伪条件。
